Overview
Stop discovering what went wrong at period end
The standard approach -- comparing a Stock Status report to the trial balance -- has five well-documented failure points that produce mismatches requiring manual investigation every period.
  • TimingTransactions after the report but before close appear in GL only
  • BackdatingPrior-period GL dates create discrepancies in both reports
  • Report DefinitionIncorrect cost method or data selection produces totals that will never agree
  • DMAAI MisconfigurationA single wrong AAI silently posts every transaction of that type to the wrong account
  • GL Class Code ChangesJDE allows changes without checking quantity on hand, stranding inventory value in the original account

Proactive Monitoring
Six Integrity Reports
A single misconfigured AAI silently routes every transaction of that type to the wrong GL account. The Integrity Reports surface configuration issues before they compound into large period-end variances.
Report 1
Model Table
Verifies DMAAI 4152 assigns a valid GL account to every GL class code. Missing entries mean items are silently excluded from reconciliation.
Report 2
DMAAI Entry Integrity
Compares balance sheet DMAAI entries against the model table. Flags object, business unit, and subsidiary mismatches.
Report 3
Excluded GL Classes
Lists GL class codes present in transactions but missing from the model table -- invisible to the reconciliation.
Report 4
Transfer Cost Variances
Lists ST/OT order pairs where the sales cost and purchase order cost do not match.
Report 5
GL Class Integrity
Compares GL class codes between Item Branch (F4102) and Item Location (F41021) records.
Report 6
Frozen Cost Integrity
Identifies items where the frozen cost in F4105 does not match the cost in recent cardex transactions.
