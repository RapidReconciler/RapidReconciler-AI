**JD Edwards - Sales Order Management**

**Reference Guide: Direct Ship Orders**

**Overview and Configuration**

**Section 1: Overview**

A direct ship order is used to record the sale of items that are purchased from a supplier who then ships the items directly to the customer. The organization does not physically handle the goods - inventory quantities and availability are not updated in JD Edwards.

When a direct ship order is created, the system generates both a **sales order** and a **purchase order** simultaneously.

Direct ship orders are entered using program **P4243**.

**Section 2: Key Setup and Configuration**

**2.1 Line Type**

The line type typically used for direct ship orders is **"D"**, which carries a **D inventory interface**. This interface instructs the system to:

- Check inventory for a valid item number.
- **Not** update on-hand quantities.
- **Not** check availability.

**2.2 Related Order Fields**

The sales order and purchase order generated by a direct ship transaction are linked through the **Related Order** fields in tables F4211 (Sales Order Detail) and F4311 (Purchase Order Detail):

- In the **sales order**, these fields are found on the detail behind the details (option 1 from a detail line).
- In the **purchase order**, these fields are found in the fold of the detail line.

**2.3 AAI Tables Involved**

The following AAI tables are invoked during direct ship order processing:

| **Side**       | **AAI**   | **Account**            |
| -------------- | --------- | ---------------------- |
| **Sales**      | 4230      | Revenue                |
| **Sales**      | 4220      | Cost of Goods Sold     |
| **Sales**      | 4240      | Inventory              |
| **Sales**      | A/R Trade | Accounts Receivable    |
| **Purchasing** | 4310      | Inventory              |
| **Purchasing** | PC        | Accounts Payable Trade |

**Section 3: Making Changes to Direct Ship Orders**

All changes to direct ship sales and purchase orders should be made through program **P4243**. The following behavior applies:

- Changes made to the **sales order** will automatically update the corresponding purchase order.
- Changes made to the **purchase order** will **not** update the sales order.

The receipts program (P4312) can be configured to update the status of the sales order upon full receipt of a direct ship purchase order line. Partial receipts will not trigger a status update.

**Section 4: Important Notes**

- **Associated text** must be entered separately on the sales order and the purchase order - it does not transfer between the two automatically.
- **Kit and configured items** cannot be entered on a direct ship order. To process kits, use the standard sales or purchase order entry programs.
- A **2-way match** can be used for the purchase order, since inventory quantities are not being updated.
- In release A7.3 (option 27) and A8.1 (option 28), additional options were added to control where the header branch and detail branch will be sourced from.

**Section 5: Direct Ship Orders and Partial Shipments**

When a supplier performs a partial shipment on a direct ship order, the following sequence must be followed:

**Critical Rule:** Always **ship confirm BEFORE receiving** against the OD document. Ensure that processing option #16 behind ship confirm is set to **blank**.

**Step-by-step process for partial shipments:**

- The supplier ships half of the order and invoices for that half.
- Ship confirm the partial quantity on the sales order side.
- Invoice the customer for the partial shipment.
- Process the partial receipt against the OD purchase order document.
- When the supplier ships the remaining quantity, ship confirm and invoice the customer **before** processing the receipt for the remaining quantity on the OD document.

**Section 6: Multi-Currency and Direct Ship Orders**

**6.1 Comparing Exchange Rates**

A processing option can be configured to compare the exchange rates on the sales order and issue a warning message if a significant currency rate change is detected.

**6.2 Viewing Orders in Foreign Currency Mode**

When viewing a direct ship order in foreign currency mode:

- The extended price fields for the sales order display decimals for the currency specified for the "Sold To" address number in the Customer Master.
- The extended cost for the purchase order portion displays using the decimals for the currency from the Supplier Master.

**6.3 Viewing Orders in Domestic Currency Mode**

When viewing in domestic currency mode, the system displays the extended detail for both the sales and purchasing sides using the decimals specified for the base currency of the company.

**6.4 Updating Currency**

Changes to the order can only be made in one currency mode at a time. The system automatically updates both foreign and domestic fields when a change is made. Updated order information can be viewed in either currency mode regardless of which mode was active when the change was made.

**Section 7: ECS Functionality with Direct Ship Orders (Release A8.1)**

**7.1 ECS Format Processing Option**

In release A8.1, processing option **#22** was added to display the ECS (Extended Customer Service) format for direct ship order entry.

**7.2 ECS Advanced Functionality**

If **Load and Delivery Management** is in use and the ECS Control has been activated in Sales Order Management system constants, the ECS format must be used to enter direct ship orders. This format includes the additional fields required by the Load and Delivery Management system.

**7.3 Agreement Numbers**

Agreement numbers can only be entered while in the ECS format. The following behavior applies:

- If the customer and item combination matches a single agreement number, the system automatically populates the agreement number on the order.
- If the combination matches multiple agreement numbers, a selection window is displayed from which the appropriate agreement can be chosen.
- If an agreement is assigned to a detail line, the system performs agreement processing for that line.
