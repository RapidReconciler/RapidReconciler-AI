Installing the Production Database

This walkthrough covers the end-to-end installation of the RapidReconciler production database, the SQL Server Integration Services (SSIS) package, and the SQL Agent job that drives nightly data loads. The work is normally completed during a single 2-hour web meeting with the customer's IT contact.

Each section is self-contained. Use the sidebar or the table below to jump to a topic. From any topic, the Back to Index button returns you here.

How to Use This Walkthrough

The walkthrough is organized as four installation tours plus reference material. Work through the tours in order; the reference sections (Architecture & Requirements, Security Requirements) can be consulted as needed. The whole procedure typically takes about 2 hours when working alongside the customer's IT contact.

Before You Start

This walkthrough assumes you are an entry-level SQL Server DBA or a developer with comparable familiarity. A quick check of skills and inputs before beginning saves rework later.

Time required

The full installation typically takes about 2 hours. Plan to do it in one sitting alongside the customer's IT contact during a scheduled web meeting.

Required Skills

RapidReconciler is a Microsoft SQL Server based product. You should be comfortable with:

  • Running scripts in SQL Server Management Studio (SSMS)
  • Basic SQL Server database configuration (creating databases, logins, users, role memberships)
  • SQL Server Agent job configuration — opening jobs, editing steps, configuring schedules
  • SQL Server Integration Services (SSIS) — opening a package in Visual Studio, working with connection managers, deploying to a catalog

If any of these is unfamiliar, pair with a senior DBA on the first install.

Required Tools on the Database Server

  • SQL Server Management Studio (SSMS)
  • Visual Studio Community with the SQL Server Integration Services extension installed
  • SQL Server Data Tools (SSDT)
  • The appropriate OLE DB provider for the customer's JDE source (covered in Architecture & Requirements)

What's in the Installation Zip

The customer's IT contact will receive a zip file from GSI containing the scripts and SSIS package needed for the install. The zip contains:

Note

The zip is downloaded by the customer's IT contact directly from GSI's distribution location and unzipped on the target database server. The exact distribution method (email, secure link, etc.) varies by customer; coordinate with the GSI sales or support contact assigned to the install.

Architecture & Requirements

This section covers the components of a RapidReconciler deployment and the minimum hardware and software requirements for each. Read this before scheduling the install with the customer — many install delays come from server requirements not being met.

Architecture Overview

RapidReconciler consists of several components. Setup is flexible and can vary based on the customer's needs. The main components are:

ComponentRole
Database serverRuns Microsoft SQL Server. Hosts the RapidReconciler database. Data is extracted from JDE using Integration Services in read-only mode.
Application serverRuns the RapidReconciler agent and data services. Can be the same machine as the database server if the customer dedicates a server to RR.
User PCEnd-user workstation. Must be connected to the company network to view data.
Version control server (VALC)GSI-hosted. Maintains licensing and control information for RapidReconciler configurations.

Data Synchronized with the RR Agent

All data stored on GSI servers employs 256-bit encryption and is limited to:

  • Public-facing IP address of the application server
  • Internal IP address of the application server
  • IP address of the database server
  • Credentials to access the RapidReconciler database
  • RapidReconciler database name(s) and port numbers
  • Usernames and passwords (GSI cannot retrieve passwords, only reset them)
  • User access security settings (companies and tabs)
  • 5-digit company number and name from JD Edwards (used for licensing only)

Internet traffic is outbound only.

Notes on deployment
  • If a server is dedicated for RapidReconciler, both the database and agent can be on the same machine.
  • Port 443 outbound is used by RapidReconciler for JMS communications.
  • Work with the customer's IT support to decide how RapidReconciler will be deployed.
  • Only 1 instance of the agent can be run on the internal network. The agent can support multiple RapidReconciler databases, even if they are on different servers.

Disk Space

Estimate disk space based on the customer's transaction volume:

  • 100 GB per 1 million cardex (F4111) records per month
  • Estimate covers 1 year of data; records may be purged after year-end close

Example: a customer generating 2,000,000 cardex records/month requires 200 GB of disk space allocated to the RapidReconciler database.

SSIS Server — When Different from the Database Server

If SSIS is run on a separate server (uncommon, but possible), that server must have:

  • Windows Server 2016 or later
  • 16 GB RAM minimum
  • SQL Server Management Studio
  • SQL Server Data Tools

JDE Source Driver Requirements

The OLE DB provider needed depends on what the customer's JDE database runs on:

OracleOracle 32-bit client (64-bit optional). Select the Administrator install option, make applicable tnsnames.ora entries; if installing both architectures, place in the same Oracle home.
SQL ServerNo additional configuration required.
Common cause of install delays

Driver mismatch (e.g. only 64-bit installed when both are needed) is a frequent cause of SSIS connection failures. Verify the OLE DB provider is installed correctly before the install meeting.

Working with a New Client

Before scheduling the install meeting, gather the information below. Missing inputs at install time are the most common reason an install meeting runs over the planned 2 hours or has to be rescheduled.

Information from the Sales Rep

RapidReconciler licensing is based on JD Edwards company numbers — normally one license per company. From the sales rep, obtain:

  • The number of companies the customer has purchased
  • The exact JD Edwards company numbers (e.g. 00001, 00002)
  • The customer name (e.g. Acme Manufacturing)
  • The customer's IT contact — email and phone number

Information from the Customer's IT Contact

Send the GSI Tech Requirements document to the customer's IT contact ahead of the install meeting. Have them complete and return the requested information from page 3, which captures:

  • Server topology (dedicated vs. separate database/application servers)
  • SQL Server version and authentication mode
  • JDE database type (AS/400, Oracle, SQL Server) and connection details
  • Read-only credentials for the JDE database (use rapidrec/rapidrec if possible)
  • Application server IP addresses (internal and external)
  • Maintenance window preferences for the SQL Agent job
Note

The Tech Requirements document is a separate file maintained by GSI. If you are unsure where to obtain the current version, contact your team lead or check the team's shared documentation location.

Schedule a 2-Hour Web Meeting

Once the Tech Requirements have been returned, schedule a 2-hour web meeting with the customer's IT contact. The agenda for that meeting is:

  1. Confirm the server(s) meet the minimum requirements (drivers, Visual Studio, SSMS, etc.)
  2. Install the RapidReconciler database, create the rruser login, and create the RapidReconciler_Prod SQL Agent job (Tour 2)
  3. Create the Integration Services Catalog (Tour 1)
  4. Configure and deploy the SSIS package (Tour 3)
  5. Perform the initial data load (Tour 4)
Tip

The order of operations during the meeting is flexible — Tour 1 (creating the SSIS catalog) can be done before Tour 2 (database install) if convenient, since they are independent. The remaining tours must be completed in the order shown above.

Tour 1: Create the SSISDB Catalog

The Integration Services Catalog (SSISDB) must exist on the target SQL Server before the RapidReconciler SSIS package can be deployed. If the catalog has already been created on this server, skip to Step 2 to create the RapidReconciler folder.

1
Tour 1 · Step 1 of 2
Create the SSISDB Catalog
  1. Open SQL Server Management Studio and connect to the target SQL Server instance.
  2. In Object Explorer, expand the server node.
  3. Right-click Integration Services Catalogs and select Create Catalog.
  4. In the Create Catalog dialog:
    • Check Enable CLR Integration if not already enabled.
    • Check Enable automatic execution of Integration Services stored procedure at SQL Server startup.
    • Enter and confirm a Password to protect the database master key.
  5. Click OK to create the catalog.
  6. Confirm SSISDB now appears under Integration Services Catalogs in Object Explorer.
Important

Creation of the SSISDB catalog requires sysadmin privileges on the SQL Server instance. The password entered here will be required if the catalog ever needs to be restored from backup — store it in a secure location.

2
Tour 1 · Step 2 of 2
Create the RapidReconciler Folder
  1. In Object Explorer, expand Integration Services Catalogs → SSISDB.
  2. Right-click SSISDB and select Create Folder.
  3. In the Create Folder dialog:
    • Set Folder name to RapidReconciler (or RR).
    • Optionally enter a description such as RapidReconciler SSIS packages.
  4. Click OK to create the folder.
  5. Confirm the RapidReconciler folder now appears under SSISDB in Object Explorer.
Note

This folder name must match the deployment path used in Tour 3, Step 6. The path /SSISDB/RapidReconciler/RapidReconciler references this folder as the first RapidReconciler segment.

Tour 2: Install the RapidReconciler Database

This tour runs the four installation scripts in sequence to create the database, install all database objects, create the rruser SQL login, and create the RapidReconciler_Prod SQL Agent job.

1
Tour 2 · Step 1 of 5
Confirm the Installation Files Are on the Server

Confirm with the customer's IT contact that the GSI installation zip has been downloaded and unzipped on the target database server. The unzipped folder should contain the four .sql scripts and the RapidReconciler-Prod.dtsx SSIS package listed in Before You Start.

If the zip is not yet on the server, pause and have the customer download and unzip it before continuing.

2
Tour 2 · Step 2 of 5
Create the Database
  1. Log on to the database server alongside the customer's IT contact.
  2. Open SQL Server Management Studio.
  3. Log in with an account that has sysadmin privileges.
  4. Open and run 1 - RapidReconciler Database Creation Script.sql.

The script will:

  • Create mdf and ldf files in the default locations specified by the server instance
  • Set the initial database size to 5 GB
  • Create the database with the default name RapidReconciler_Prod
Important — Change the database context

After this script completes, change the database context from master to RapidReconciler_Prod in the SSMS connection dropdown before opening any of the remaining scripts. Each subsequent script must run against RapidReconciler_Prod, not master.

3
Tour 2 · Step 3 of 5
Add Database Objects

Run 2 - RapidReconciler Database Object Script 178.sql against RapidReconciler_Prod.

Wait for the Successfully Completed message before continuing. This script creates all tables, views, and stored procedures the application requires.

4
Tour 2 · Step 4 of 5
Create the SQL User ID

Run 3 - RapidReconciler User Creation Script.sql.

This creates the rruser SQL login. These are the credentials the application uses to read and write data to the SQL database. The script handles both the server-level login creation and the database-level user mapping.

Permissions detail is covered in Security Requirements.

5
Tour 2 · Step 5 of 5
Create the SQL Agent Job

Run 4 - RapidReconciler SQL Agent Job Creation Script.sql.

This creates the RapidReconciler_Prod SQL Agent job. The job's package path will be configured in Tour 3 once the SSIS package is deployed, and the job's schedule will be enabled in Tour 4.

Verify before continuing

In SSMS Object Explorer, expand SQL Server Agent → Jobs. Confirm the RapidReconciler_Prod job appears in the list. If it does not, re-check that the script ran without errors.

Tour 3: Configure & Deploy the SSIS Package

This is the longest tour. You'll create a Visual Studio solution, add the prebuilt SSIS package, configure its connection managers and variables, deploy the project to the SSISDB catalog, and wire the deployed package up to the SQL Agent job created in Tour 2.

Prerequisites

  • Visual Studio Community with the SQL Server Integration Services extension installed
  • The RapidReconciler-Prod.dtsx file extracted from the installation zip
  • The SSISDB catalog and RapidReconciler folder created in Tour 1
  • The database, rruser login, and SQL Agent job created in Tour 2
1
Tour 3 · Step 1 of 8
Gather Configuration Information

Before opening Visual Studio, gather the following from the Tech Requirements document the customer's IT contact filled out:

JD Edwards Connection Details

  • Name or IP address of the JDE data server
  • Read-only credentials for the JDE database (use rapidrec/rapidrec if possible)
  • JDE server type: iSeries, Oracle, or Microsoft SQL Server

JDE Data Configuration

  • Table qualifier for JDE data (e.g. proddta. — note the trailing period)
  • Decimal places for: extended cost, unit cost, quantity on hand, transaction quantities
  • Customer's fiscal year start date (used to compute the aaStartDateGr variable — see Step 5)
2
Tour 3 · Step 2 of 8
Create the Visual Studio Solution
  1. Launch Visual Studio Community. Run as administrator if possible to avoid permission issues during deployment.
  2. Select Create a new project.
  1. Search for and select Integration Services Project, then click Next.
  1. Set the Project name to RapidReconciler.
  2. Choose an appropriate location on the server to save the solution.
  3. Click Create.
  1. Once the solution loads, confirm the project appears in Solution Explorer as RapidReconciler.
3
Tour 3 · Step 3 of 8
Add the Existing SSIS Package
  1. In Solution Explorer, expand the RapidReconciler project and right-click the SSIS Packages folder.
  2. Select Add Existing Package.
  3. In the Add Copy of Existing Package dialog, set Package location to File system, then browse to where the installation zip was extracted.
  4. Select RapidReconciler-Prod.dtsx and click OK.
  5. Confirm the package appears under the RapidReconciler project in Solution Explorer.
  6. Double-click the package to open it in the designer and verify the control flow and variables load.
  7. If prompted to upgrade the package, click OK to allow Visual Studio to update the package format.
Connection errors are expected at this stage

The package may show connection errors when it first opens — this is normal. The connection managers will be updated in Step 4. Passwords also need to be re-entered after deployment.

4
Tour 3 · Step 4 of 8
Configure the Connection Managers

Connection managers appear at the bottom-center of the package designer.

JDE Source Connection

  1. Double-click the JDE source connection.
  2. Update the Server name or IP address to the JDE data server.
  3. Enter the database credentials gathered in Step 1 (rapidrec/rapidrec if available).
  4. Click Test Connection to verify connectivity to the JDE data source.
  5. Click OK to save changes.

RapidReconciler Destination Connection

  1. Double-click the RapidReconciler destination connection.
  2. Set the Server name to the database server hosting RapidReconciler_Prod.
  3. Select RapidReconciler_Prod from the database dropdown.
  4. Enter the rruser credentials created in Tour 2.
  5. Click Test Connection to verify connectivity to RapidReconciler_Prod.
  6. Click OK to save changes.
5
Tour 3 · Step 5 of 8
Configure the Variables
  1. Open the Variables window. If not visible, go to View → Other Windows → Variables.
  2. Reference Figure 1 (Step 4) for the variable names — they appear highlighted in yellow.
  3. Update each variable using the values gathered in Step 1:
VariableValue to Enter
aaStartDateGrStart date for data extraction. Set to 2 months prior to the current fiscal year.
dbownerTable qualifier for JDE data (e.g. proddta.). Ensure there is a period at the end.
DecExtCostDecimal places for extended cost: 1 followed by the number of zeros for ECST.
DecUnitCostDecimal places for unit cost: 1 followed by the number of zeros for UNCS.
DecQTYDecimal places for quantity on hand: 1 followed by the number of zeros for PQOH.
DecQtyCXDecimal places for transaction quantities in cardex: 1 followed by the number of zeros for TRQT.
  1. Save all changes by clicking File → Save All or pressing Ctrl + Shift + S.
Common mistake

Forgetting the trailing period on dbowner (e.g. entering proddta instead of proddta.) is the single most common cause of "package deployed but extracts no data" tickets. Double-check before saving.

6
Tour 3 · Step 6 of 8
Deploy the Project to the SSIS Catalog
  1. In Solution Explorer, right-click the RapidReconciler project and select Deploy.
  2. The Integration Services Deployment Wizard opens. Click Next.
  3. On the Select Destination page:
    • Set Server name to the target SQL Server instance.
    • Connect using Windows Authentication.
    • Set Path to /SSISDB/RapidReconciler/RapidReconciler.
  4. Click Next, review the summary, then click Deploy.
  5. Wait for all deployment steps to show a Passed status.
  6. Click Close when complete.
Authentication requirement

Windows Authentication is required for deployment to the SSIS catalog. Ensure the account you are logged in with has sysadmin or ssis_admin privileges on the target SQL Server instance before proceeding.

7
Tour 3 · Step 7 of 8
Verify the Deployment
  1. Open SSMS and connect to the target server.
  2. Expand Integration Services Catalogs → SSISDB → RapidReconciler.
  3. Confirm the deployed RapidReconciler project appears and is accessible.

If the project is missing, re-run the deployment wizard. Common causes: insufficient permissions on SSISDB, or the path was set incorrectly in Step 6.

8
Tour 3 · Step 8 of 8
Configure the SQL Agent Package Path

The SQL Agent job created in Tour 2 needs to be pointed at the deployed SSIS package.

  1. In SSMS, navigate to SQL Server Agent → Jobs → RapidReconciler_Prod.
  2. Right-click the job and select Properties.
  3. Click Steps in the left panel.
  4. Edit step 1 to point to the deployed SSIS package path in the SSISDB catalog (use the path /SSISDB/RapidReconciler/RapidReconciler/RapidReconciler-Prod.dtsx or browse to the package via the SSIS Catalog browser).
  5. Click OK to save the step.
  6. Click OK to save the job properties.

Tour 4: Perform the Initial Data Load

The database, SSIS package, and SQL Agent job are all in place. The final tour enables the job's schedule and triggers the first run, which populates RapidReconciler_Prod with the customer's JDE data.

Tour 4 · Step 1 of 6
Open SQL Server Agent
  1. Launch SSMS and connect to the SQL Server instance.
  2. In Object Explorer, expand the server node.
  3. Expand SQL Server Agent.
  4. Expand Jobs.
2
Tour 4 · Step 2 of 6
Open the Job Properties
  1. Right-click the job RapidReconciler_Prod.
  2. Select Properties from the context menu.
  3. The Job Properties dialog opens.
3
Tour 4 · Step 3 of 6
Navigate to the Schedules Tab
  1. In the left panel of the Job Properties dialog, click Schedules.
  2. You will see a list of schedules currently associated with the job.
4
Tour 4 · Step 4 of 6
Enable the Job Schedule
  1. Select the schedule from the list and click Edit.
  2. In the Job Schedule Properties dialog, check the Enabled checkbox at the top.
5
Tour 4 · Step 5 of 6
Update the Run Time
Scheduling guidance

The RapidReconciler job should run during off-peak hours (typically overnight) to minimize impact on system performance and provide the most accurate snapshot. Coordinate with the customer's IT contact to determine the best time. The initial load may take longer than subsequent runs.

  1. Under Daily frequency, update the Occurs once at or Occurs every time fields to the agreed run time.
  2. Adjust the Start date and End date in the Duration section if needed.
  3. Click OK to save the schedule changes.
6
Tour 4 · Step 6 of 6
Save the Job
  1. Click OK in the Job Properties dialog to apply all changes.
  2. The job schedule is now enabled and will run at the next scheduled time.
Note

Schedule changes take effect at the next scheduled run time and will not interrupt a currently running job. After the initial load completes, the schedule can be adjusted as needed to meet the customer's data refresh requirements.

Security Requirements

This section is the permissions reference for the install. Most of these permissions are set by the scripts in Tour 2, but they are documented here for verification, troubleshooting, and any cases where manual configuration is required.

SQL Server Login and User Permissions

The rruser SQL login must be configured with the following permissions:

ObjectAccountPermission
RapidReconciler_Prod databaserruserdb_datareader
RapidReconciler_Prod databaserruserdb_datawriter
msdbrruserSQLAgentOperatorRole

Configuring rruser Manually

  1. In SSMS, navigate to Security → Logins and open the rruser login properties.
  2. Under User Mapping, select RapidReconciler_Prod and assign db_datareader and db_datawriter.
  3. Also map to msdb and assign SQLAgentOperatorRole.
  4. Click OK to save.

SQL Server Agent Service Account

The SQL Server Agent service account is configured separately from rruser. Ensure:

  • The service account has db_datareader and db_datawriter on RapidReconciler_Prod.
  • The service account has Log on as a service rights on the server running the agent job.
  • If a proxy account is used for job steps instead of the service account, the proxy must be explicitly granted access to the SSIS subsystem under SQL Server Agent → Proxies.

SSIS Catalog Permissions

The account running the SQL Agent job must have access to execute the SSIS packages in the catalog:

RoleAccountPurpose
ssis_adminSQL Agent service account or proxyFull administrative access to the SSIS catalog
ssis_logreaderSQL Agent service account or proxyRead access to execution logs for troubleshooting
dc_operatorSQL Agent service account or proxyPermission to execute packages
READ on SSIS EnvironmentSQL Agent service account or proxyRequired if environments are used to store connection credentials

Configuring SSIS Catalog Permissions

  1. In SSMS, expand Integration Services Catalogs → SSISDB.
  2. Right-click SSISDB and select Properties.
  3. Click Permissions and add the SQL Agent service account or rruser.
  4. Grant dc_operator at minimum to allow package execution.
  5. Grant ssis_logreader to allow log visibility for troubleshooting.
  6. If SSIS environments are used to store connection-manager parameters, grant READ on the environment and ensure it is referenced in the package execution properties.
  7. Click OK to save.
Proxy account note

If the SQL Agent job steps run under a proxy account, the proxy must also be granted access to any network shares or UNC paths used by the SSIS package, in addition to the SSIS catalog permissions above.

What to Do Next

The database is installed, the SSIS package is deployed and configured, and the SQL Agent job is scheduled. The first scheduled run will populate RapidReconciler_Prod with the customer's JDE data.

Verify the Initial Load

After the first scheduled run completes, verify the load succeeded:

  1. In SSMS, expand SQL Server Agent → Jobs and right-click RapidReconciler_Prod.
  2. Select View History.
  3. Confirm the most recent run shows a green checkmark and a non-zero duration.
  4. If the run failed, expand the failed step in the history view to see the error message — the agent log files in the SSISDB catalog also have detailed execution logs.

Set Up the Client in VALC

Once the database installation is verified, the customer needs to be set up in VALC (the Version Control Server). VALC setup is a separate procedure handled by the GSI tech team and includes:

  • Creating the client record in VALC and registering the database
  • Creating the initial GSI Admin user account
  • Configuring modules and additional fields per the customer's contract
  • Coordinating the DNS/Cloudflare reference for the customer's .getgsi.com URL
  • Installing the RapidReconciler agent on the customer's application server
  • Licensing the customer's company numbers
  • Handing off login credentials to the customer's RapidReconciler administrator

The VALC setup procedure is documented separately. Hand the customer's information off to the GSI tech team responsible for the VALC side of the install.

You're done

The database side of the install is complete. The customer is one step closer to going live with RapidReconciler.
