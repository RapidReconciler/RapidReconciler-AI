Overview

How RapidReconciler SSL Works

The RapidReconciler user interface is secured using a wildcard SSL certificate (*.getgsi.com) provisioned and managed by GSI. This eliminates the need for individual customers to purchase, configure, or renew their own SSL certificates, reducing both cost and administrative overhead.

Each customer is assigned a unique domain URL (e.g. rrprod-customername.getgsi.com) configured in VALC during initial installation. GSI manages the DNS and certificate infrastructure required to make this URL resolve securely to the customer's server.

DNS Configuration

DNS 'A' Record Management

GSI I/T maintains DNS records with the GSI Internet Service Provider (ISP). For each RapidReconciler customer, an 'A' record maps the customer's assigned domain URL to the internal IP address of their application server.

Example A Record
Domain URLInternal IP Address
rrprod-acmemanufacturing.getgsi.com192.168.1.100

When a New 'A' Record Is Required

A new or updated 'A' record is needed in the following situations:

SituationAction Required
New customer being set upAdd new 'A' record
Customer migrates to a new serverUpdate 'A' record with new IP
Customer's internal IP address changesUpdate 'A' record with new IP

Process

  1. Confirm the domain URL and internal IP address on the Client Details page in VALC.
  2. Submit the request to GSI I/T with the domain URL and internal IP address.
  3. GSI I/T adds or updates the 'A' record with the ISP.
  4. Allow time for DNS propagation before testing connectivity.
Note

DNS propagation can take anywhere from a few minutes to several hours depending on the ISP and TTL settings. The RapidReconciler UI will not be accessible at the domain URL until propagation is complete.

Certificate Renewal Process

The *.getgsi.com wildcard certificate covers all subdomains under getgsi.com — a single certificate secures every customer's RapidReconciler URL. GSI is responsible for procuring and renewing it.

1
Backend Update — UI Developer
The UI developer updates the backend configuration with the renewed certificate. This ensures the RapidReconciler web interface continues to serve the correct certificate after renewal.
2
Agent Deployment via VALC
Once the backend is updated, the renewed certificate must be deployed to each customer's RapidReconciler Agent via the Manage Deploys process in VALC. This pushes the updated agent package to each customer server.
3
Verification
After deployment, confirm the RapidReconciler UI loads correctly and the browser shows a valid, non-expired certificate for the customer's domain URL. Check that the issuer and expiry date reflect the renewed certificate.
Important

If the certificate is not deployed to the agent before the existing certificate expires, customers will receive browser security warnings and may be unable to access the RapidReconciler UI. Plan renewals well in advance of the expiry date.

Responsibilities

Who Does What

GSI
  • Procure and renew the wildcard SSL certificate
  • Configure the domain URL in VALC
  • Deploy updated agent via VALC Manage Deploys
  • Verify connectivity and certificate validity post-renewal
GSI I/T
  • Add or update DNS 'A' records with the ISP
  • Coordinate on IP address changes
  • Resolve DNS propagation issues beyond 24 hours
GSI UI Developer
  • Update backend configuration after certificate renewal
  • Confirm web interface serves the renewed certificate correctly
GSI Staff
  • Submit DNS update requests with correct domain URL and IP
  • Trigger Manage Deploys after backend is confirmed updated
  • Perform post-deployment browser verification per customer
