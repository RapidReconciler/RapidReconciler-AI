Before You Start

This walkthrough is designed to live beside a live VALC window. A few quick things to set up first so the experience flows.

Setup Checklist

  1. Open VALC in a separate browser window or tab.
  2. Sit it side-by-side with this document if you have screen space — half-and-half works well.
  3. Pick an existing customer to look at as you go. Choose one with a healthy install (Status badge says Connected). Ask a senior tech for a recommendation if you're not sure which to use.
  4. Have the reference doc handy — there will be pointers throughout this walkthrough to specific sections of it.
Read-Only Tour

You will not create, edit, or delete anything during this walkthrough. When the tour opens popups like Add Client or Edit Database, you'll close them without saving. The goal is recognition, not action.

What Each Tour Looks Like

Each tour breaks into a series of numbered steps. Each step tells you what to look at or click on, and points out the subtle things that are easy to miss but important to know.

Take your time. Pause anywhere. Come back tomorrow if you need to. The walkthrough won't go anywhere.

Tip

If a step references a feature in the reference doc that you'd like to read about in depth, follow the link. The walkthrough establishes context; the reference doc has the field-level detail.

Tour 1: Main Navigation

You've just logged into VALC. Look at the left side of the screen. This tour orients you to the main navigation sidebar — what each item is for, and which ones you'll actually use day-to-day.

1
Step 1 of 2
Find the Main Navigation

Look at the left sidebar. You should see five items: Clients, SQL Scripts, Message Users, Key Generator, and Genius.

Of those five, you'll spend ~95% of your time on Clients. The others are for specialized situations:

  • SQL Scripts is developer-only — used to apply update scripts across customer databases. Not part of your install workflow.
  • Message Users broadcasts maintenance notices to all RR users across all clients.
  • Key Generator and Genius are rarely used by tech.
Why this matters

A common mistake new techs make is poking around in SQL Scripts thinking it's part of the install workflow. It's not — knowing what to ignore is half the battle.

2
Step 2 of 2
Click Clients

Click the Clients item in the sidebar. The main panel will load the Clients grid.

The Clients grid is your home base. Everything install-related starts here. You'll explore it in the next tour.

Tour 2: The Clients Grid

You're now looking at a list of every customer GSI has set up in VALC. Each row is one customer. This tour walks you through what's on the grid and how to navigate from it.

1
Step 1 of 4
Read the columns

Read the column headers from left to right: Status, Name, Services, Agent, System Status, System Status Message, License End, Notes, Options.

Two columns are called "status" — they mean different things

The leftmost Status column is whether the agent is connected to VALC right now (green Connected / red Disconnected). The System Status column further right is whether the most recent data sync from JDE succeeded.

These two badges fail in different ways and require different troubleshooting paths. Mixing them up sends you down the wrong rabbit hole.

2
Step 2 of 4
Look at the top-right of the grid

Find the dropdown that says Actives and the blue Create Client button.

  • The Actives dropdown filters which customers you see. Other filter values surface inactive or archived clients.
  • Create Client is how new customers get added — you'll look at that form in Tour 4.
3
Step 3 of 4
Hover on the Options column

Find any row and hover on the icons in the rightmost Options column. You should see two icons: one labeled Manage Clients and one labeled Manage Deploys.

Why this matters

Manage Clients is where you go to do anything for that customer — review their setup, edit their database, change their license, etc. It's by far the most-used button in the whole app for tech.

4
Step 4 of 4
Open one customer's Manage Clients

Pick a healthy active customer (their Status badge says Connected). Click the Manage Clients icon for that row. You'll be taken to their detail screen, which Tour 3 walks you through.

Reference: Reference doc Part 2 Step 3 (Verify Connectivity in VALC) shows what a healthy row should look like after an install completes.

Checkpoint

Before moving on, can you answer in your own words?

  • What's the difference between the leftmost Status column and the System Status column?
  • What does the Actives filter hide?
  • If you wanted to do anything for a specific customer, which icon would you click?

If any of those is fuzzy, scroll back up and re-read. The rest of the tour assumes you've got these.

Tour 3: The Manage Client Tabs

You're now looking at the Manage Client screen for the customer you picked. There are four tabs across the top: Client Details, Databases, Companies, User Accounts. You're landed on Client Details.

Tab 1: Client Details

The customer's master configuration record. Most install and customer-request work touches this tab at some point.

1
Tab 1 · Step 1 of 4
Read the top section

Look at the upper portion: Client Name, Domain URL, Setup Step, Agent Internal/External IP, License dates, Version, etc.

Watch for

The Setup Step field is critical and easy to miss. It tracks where the customer is in their lifecycle — values like Created, Connected, etc. The agent install requires this to be on Created before it'll prompt for download.

"Setup Step is wrong" is the explanation behind a lot of "the install isn't working" tickets.

2
Tab 1 · Step 2 of 4
Look at the toggles

Scroll down to the toggle section. You'll see toggles for Active, Password Policy, HTTPS, and various module flags.

The Password Policy toggle is the one customers most often request to be flipped after install. When ON, that customer's users are subject to the complex password rules.

Reference: Reference doc Password Policy covers what's in scope when this toggle is on.

3
Tab 1 · Step 3 of 4
Look at the modules section

Find the modules list (usually below the toggles). Each module has its own Active checkbox.

These modules are licensed per the customer's contract — only check the ones they've paid for. Over-enabling modules is a billing/compliance issue.

4
Tab 1 · Step 4 of 4
Find the IP fields

Locate Agent Internal IP Address and Agent External IP Address.

Important distinction

These are the customer's application server IPs (where the RR Agent runs). The database server IP is recorded separately on the Databases tab — even if it's the same physical machine. This split trips up new techs constantly.

Reference: Reference doc Part 1 Step 3 has the full field-by-field reference for Client Details. The Overview's Discovery Call section explains the topology question that drives where each IP goes.

Tab 2: Databases

Click the Databases tab.

1
Tab 2 · Step 1 of 3
Read the rows

Each row represents one database registered for this customer. Look at the columns: Status, Revision Number, Host, Name, Category, RAM, Server Port, System Status, Options.

Most customers have just one row: RapidReconciler_Prod. Some have more (Test, Dev, Train) per their contract. Each one is a separate registration.

2
Tab 2 · Step 2 of 3
Look at the Status column

Hopefully every row shows a green Online badge. If you see something else, that's the failure-state pattern shown below.

Watch for

A database showing anything other than Online is the #1 reason "the customer can't see their data." This is the first place to check when a customer reports an issue.

3
Tab 2 · Step 3 of 3
Hover the Options icon

On any row, hover the Options icon in the rightmost column. You'll see an edit icon — that's how you open the Edit Database popup, which Tour 5 covers.

Reference: Reference doc Part 3 Step 1 has the full Edit Database field reference. Reference doc Troubleshooting has a full breakdown of Unknown / Stopped status, with remediation steps.

Tab 3: Companies

Click the Companies tab.

1
Tab 3 · Step 1 of 2
Look at the dropdown

There's a database dropdown at the top-left. If the customer has multiple databases, you'd switch between them here.

The list of company numbers below the dropdown changes based on which database is selected. Companies are licensed per database.

2
Tab 3 · Step 2 of 2
Look at the Licensed checkboxes

Scan the rightmost Licensed column. Some company numbers are checked (licensed), others not.

Only checked company numbers will appear in that customer's RapidReconciler interface. The list of available companies is populated automatically from the customer's JDE — you can only license what's there. If the list is empty even though the database is Online, there's a sync issue.

Reference: Reference doc Part 3 Step 3 covers the licensing process during a fresh install. Reference doc Troubleshooting > Companies Tab is Empty covers the "Online but no companies showing" scenario.

Tab 4: User Accounts

Click the User Accounts tab.

1
Tab 4 · Step 1 of 1
Browse but don't change anything

Look at the list of users. You'll see the GSI Admin user we set up at install plus any users the customer's admin has added themselves.

Note

Once installed and handed off, this tab is mostly the customer's responsibility. They use it to add their own team members. GSI tech only touches it when there's a specific support request — for example, the customer's admin loses access and needs a temporary password reset.

Reference: Reference doc Password Policy > Locked or Inaccessible Accounts covers when GSI gets pulled into User Accounts work.

Checkpoint

Before moving on, can you answer:

  • Which tab tells you whether the customer's database is reachable?
  • Which tab tells you which company numbers the customer is paying for?
  • Which tab is mostly the customer's job after handoff?
  • Where does the application server IP go vs. where does the database server IP go?

Tour 4: The Add Client Popup

Now we'll look at the form used to create a new customer. You will not actually save anything. Click around, then close the popup.

1
Step 1 of 2
Open Add Client

Go back to the Clients grid (use the navigation or sidebar) and click the blue Create Client button at the top-right. The Add Client popup will appear.

Notice

There's no Domain URL field in this popup. That gets added later when GSI I/T creates the DNS record. The Add Client form only captures the basics needed to create the record.

2
Step 2 of 2
Close without saving

Click the X button to close the popup. Do not click Confirm.

This was just a look at the form. Real Add Client work happens in Part 1 of the install workflow. You'll get there in the install practice phase.

Reference: Reference doc Part 1 Step 1 has the field-by-field guide for filling out Add Client.

Tour 5: The Edit Database Popup

Last popup. Same rule: no saving. We'll open it, look around, and close.

1
Step 1 of 2
Open Edit Database

Go back to the customer you were looking at earlier (Manage Clients). Click the Databases tab. On any row, click the edit icon in the Options column. The Edit Database popup appears.

Three things to notice
  1. Address is the database server's IP (not the application server's, even if they're the same machine). Server Port (Don't change) is an internal port — different from the SQL port.
  2. Category values: Production, Test, Archive, Change Control, Inactive. This doesn't change behavior — it's purely a filter for the developer-only SQL Scripts page.
  3. Max Memory options: 2, 4, 6, 8, 12 GB. Controls RAM allocated on the customer's application server (not database). More users = more RAM. Default is 2 GB; bump only if performance warrants and the customer's machine has the headroom.
2
Step 2 of 2
Close without saving

Click X to close. Do not click Confirm.

Reference: Reference doc Part 3 Step 1 has the full Edit Database field reference, including the backup-restore approach for adding a second or third database without re-running creation scripts.

What to Do Next

You've now completed the read-only tour. You've seen:

  • The Main Navigation and what to ignore vs. focus on
  • The Clients grid and the two different "status" indicators
  • The four Manage Client tabs (Client Details, Databases, Companies, User Accounts)
  • The Add Client popup and the Edit Database popup

You haven't yet done a real install — that's intentional. This walkthrough was about building familiarity with what you're looking at, so when you go to do the install for real, the screens are recognizable.

Three Steps to Graduate

1
After this tour
Talk to a senior tech

Walk them through what you saw, in your own words. They'll catch gaps in your understanding before they cost you on a real install.

2

Parts 1, 2, 3 in order. Now that you've seen the screens, the steps will read more concretely.

3
When the next install lands

The reference doc is your active companion during it. Use the agent log files as your safety net when something doesn't behave as expected.
